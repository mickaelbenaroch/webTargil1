Welcome To My AmazonBooks Categories By Events

Introduction

In this tutorial you will learn about my API functionnality.
Don't worry it's very easy.
This API allows you to find Categories of books.
The Json file contains 3 categories of sort of books: recommended books, notsold books and bestseller books.
Each category has a few properties, in particular has an array of subcategories that contains books by genre.
For example, in the best seller category, we have subcategories like that each one has an id (e.g "89"), a subject (e.g "action"), a status (e.g "A") and finally a number of books (e.g "50000");
All you have to do is to make a http get/post valid request to get informations.



API Http Requests explanation

  • https://targil1part1.herokuapp.com/categories

    • This route is a http GET request that will return you all the categories that are available.
    Doesn't require ANY PARAMETER

  • https://targil1part1.herokuapp.com/categories/:category_id

    • This route is a http GET request that will return you a specific category by ID search.
    Here you must pass ONE QUERY STRING PARAMETER: "category_id"

  • https://targil1part1.herokuapp.com/subcategories/:status/:subject

    • This route is a http GET request that will return you a specific category that one of its subcategories 'status' is the status parameter that you sent and that its 'subject' is the subject parameter that you send in query string.
    Here you must pass TWO QUERY STRING PARAMETERS: "status" + "subject"

  • https://targil1part1.herokuapp.com/categorybyname

    • This route is a http POST request that will return you a specific category that its name is the name parameter that you sent in query string.
    Here you must pass A JSON IN BODY REQUEST e.g :
    "categoryname": "[name of the wanted category]"
    Don't forget Its a POST HTTP REQUEST

Debbuging

You made request and encontered troubles?
1 - CHECK THE ROUTE, it has to be identical to the route you choose in this page.
2 - CHECK THE QUERY STRING PARAMETERS that you are sending, if you are using postman the key of the parameters has to be identical to those in the specific route explanation.
3- CHECK THE BODY REQUEST, if you are doing a post request, send exactly what the route requires.
